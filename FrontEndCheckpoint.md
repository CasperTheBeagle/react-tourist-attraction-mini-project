### Required **Requirements (Requirement ขั้นต่ำที่ต้องทำส่ง)**
1. ใช้งานสามารถพิมพ์คำค้นหาลงในช่อง Input เพื่อค้นหาสถานที่ท่องเที่ยว
- ถ้าเปิด Website ขึ้นมาแล้ว User ยังไม่ได้พิมพ์ข้อความลงใน Input ให้เอาข้อมูลจาก Server ทั้งหมดมาแสดงได้เลย
2. ผู้ใช้งานจะต้องเห็นผลลัพธ์ที่ค้นหาออกมาเป็นรายการของสถานที่ท่องเที่ยวที่เกี่ยวข้องกับคำค้นหาแบบในรูป UI
- ผลลัพธ์รายการสถานที่ท่องเที่ยวที่แสดงบน UI จะต้องไปขอข้อมูลจาก Server
- รายละเอียด Description จะต้องแสดงไม่เกิน 100 ตัวอักษร
- ผลลัพธ์รายการสถานที่ท่องเที่ยวที่แสดงบน UI จะต้องแสดง
- Title
- Description
- รูปภาพใหญ่ 1 รูป (โดยที่รูปภาพใหญ่เอามาจาก Value ที่อยู่ใน Array อันแรกของ Key photos ที่อยู่ใน Response จาก Server)
- รูปภาพประกอบรูปเล็ก 3 รูป
3. เมื่อ User คลิกปุ่ม “อ่านต่อ” จะต้องเปิดลิ้งค์ขึ้นมาใน Tab ใหม่ของ Web Browser ลิ้งค์จะเอามาจาก Property `url`
เมื่อ User คลิกปุ่ม “อ่านต่อ” จะต้องเปิดลิ้งค์ขึ้นมาใน Tab ใหม่ของ Web Browser ลิ้งค์จะเอามาจาก Property url
4. ชื่อของสถานที่ท่องเที่ยวแต่ละรายการจะต้องเป็นลิ้งค์ที่สามารถกดเข้าไปดูรายละเอียดเพิ่มเติมได้ ซึ่งลิ้งค์นี้จะเปิดหน้าเว็บไซต์ใหม่ขึ้นมา
- ลิ้งค์ของสถานที่ท่องเที่ยวนั้นๆ เอามาจากข้อมูลที่ Server ส่งกลับมาให้ใน Object จาก Response ที่มี Property ชื่อว่า url
5. สถานที่ท่องเที่ยวแต่ละรายการจะต้องแสดงหมวดหมู่
- ซึ่งข้อมูลหมวดหมู่เอามาจากข้อมูลที่ Server ส่งกลับมาให้ใน Object จาก Response ที่มี Property ชื่อว่า tags
6. เขียนโค้ดให้มี Quality ตามหลักการต่อไปนี้
- DRY (Don’t Repeat Yourself)
- หลีกเลี่ยงการเขียนโค้ดซ้ำๆ โดยแยกส่วนที่เหมือนกันออกมาเป็นคอมโพเนนต์ที่นำกลับมาใช้ได้ (Reusable)
- High Cohesion
- จัดโครงสร้างไฟล์และโฟลเดอร์ตามหน้าที่ที่เกี่ยวข้องกัน เพื่อให้อ่านและดูแลรักษาง่าย
- Loose Coupling
- ลดการพึ่งพาระหว่างส่วนต่าง ๆ ของระบบ เพื่อให้แก้ไขหรือปรับปรุงได้โดยไม่กระทบส่วนอื่นมาก
- Single Responsibility Principle (SRP)
- แต่ละ Module หรือ Component ควรมีหน้าที่เดียว ทำให้โค้ดชัดเจนและง่ายต่อการดูแล
### Optional **Requirements (Requirement โจทย์เสริม)**
1. เมื่อ User คลิกที่หมวดหมู่จะต้องเอาข้อความที่คลิกไปใส่ลงในช่อง Input เพื่อทำการค้นหา
- เมื่อคลิกหมวดหมู่อีกครั้ง ก็จะเอาคำไปต่อจากคำเดิมในช่อง Input เช่น ถ้าคลิก หมวดหมู่ เกาะ และคาเฟ่ ข้อความที่อยู่ในช่อง Input จะเป็นแบบนี้ “เกาะ คาเฟ่”
- ถ้าคลิกที่หมวดหมู่เดิมแสดงผลในช่อง Input อยู่แล้ว จะต้องไม่แสดงผลหมวดหมู่เดิมซ้ำ 2 ครั้ง
- เช่น ใน Input จะ*ไม่แสดงผล*แบบนี้ “เกาะ คาเฟ่ เกาะ”
2. ในแต่ละรายการสถานที่ท่องเที่ยวจะมี ปุ่มสีฟ้าๆ ที่สามารถกดแล้วจะ Copy ลิ้งค์ที่สามารถกดเข้าไปดูรายละเอียดเพิ่มเติมได้ลงใน Clipboard @ex_clipboard.text
### Server Information
- เมื่อ Clone ลงมาจะต้องพิมพ์ Command cd ไปที่ Folder server ก่อน แล้วพิมพ์ Command npm install เพื่อติดตั้ง Package ที่จำเป็นในการ Start ตัว Server
- จากนั้นจะ สามารถ Start ตัว Server ได้ด้วย Command npm start
- กำหนดให้ Server มีข้อมูลดังนี้
- Endpoint ของ Server คือ http://localhost:4001
- Server ได้กำหนด API ไว้ดังนี้
| **HTTP Methods** | **Endpoint** | **Description** |
| --- | --- | --- |
| GET | <endpoint>/trips?keywords=<คำค้นหา> | API ที่เอาไว้ค้นหาสถานที่ท่องเที่ยวที่เกี่ยวข้องกับ Keywords |
ตัวอย่างข้อมูลจาก API ด้านบน @ex_json_api.text
### How to Deliver High Quality Codebase (วิธีส่งมอบโค้ดที่มีคุณภาพแบบมือโปร)
"แนะนำให้ทำตามแนวทางด้านล่าง เพื่อส่งมอบงานที่มีคุณภาพที่ใกล้เคียงกับการทำงานจริง"
1) ให้ใช้คำศัพท์สื่อสารใน Code ให้ชัดเจน เช่น
- การตั้งชื่อ React Component
- การตั้งชื่อ className (ถ้าใครเขียน CSS แบบแยกไฟล์)
- การตั้งชื่อ Function
2) ให้แบ่งและสร้าง React Component ให้เหมาะสมกับ UI ของเว็บไซต์ที่กำหนดไว้
3) เขียนโค้ดให้มี Quality ตามหลักการต่อไปนี้
- DRY (Don’t Repeat Yourself)
- High Cohesion
- Loose Coupling
- Single Responsibility Principle (SRP)
4) ให้แตก Branch ตามหลัก Feature based branch คือ
- มี Branch main ไว้เก็บโค้ดที่พัฒนาเสร็จและพร้อมใช้งาน
- มี Branch dev ไว้ใช้ทดสอบโค้ดก่อน Merge เข้า main
- มี Branch feature/name เช่น feat/landing-page ไว้สำหรับพัฒนาฟีเจอร์
5) ให้เขียน Commit Message อย่างเป็นระบบ โดยเขียนในรูปแบบ <type>/<scope>: <details> เช่น feat: add search function in landing page
